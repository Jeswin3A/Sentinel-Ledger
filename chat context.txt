# Context: Real-Time Fraud & Compliance Observability Engine

## Objective
Build a production-grade, event-driven Full-Stack + AI application that monitors financial transactions in real-time, surfaces anomalies on a live Angular dashboard, and uses an AI agent to generate automated compliance reports. The entire stack must be built using 100% free-tier services.

## Architecture & Workflow
1. **Transaction Ingestion (ASP.NET Core):** A high-throughput REST API receives incoming transactions and pushes them into an Upstash Redis Queue.
2. **Event Processing & Evaluation (Python/FastAPI):** A lightweight background service processes events from Redis, runs deterministic fraud-rule checks, and saves the data to a Neon/Supabase PostgreSQL database.
3. **AI Agent Enrichment (Gemini API):** For high-risk transactions, the Python backend invokes a structured Gemini prompt to analyze the transaction context against compliance standards and outputs a JSON mitigation strategy.
4. **Real-Time Streaming (WebSockets):** The ASP.NET Core backend broadcasts transaction alerts and AI insights to the frontend.
5. **Live Dashboard (Angular):** A highly responsive UI displaying real-time metric counters, transaction logs, and an interactive compliance review panel.

## Core Feature Roadmap

### Phase 1: Database & Event Broker Setup
- Provision a free PostgreSQL instance on Neon or Supabase.
- Spin up a free Redis instance on Upstash.
- Define schemas for `Transactions` (Id, Amount, Timestamp, Status, RiskScore) and `ComplianceAudits` (Id, TransactionId, AIAnalysis, Status).

### Phase 2: ASP.NET Core Ingestion & WebSocket Gateway
- Build an endpoint `POST /api/transactions` that validates payloads and publishes them to Redis.
- Implement a WebSocket or SignalR Hub to broadcast live transaction state updates to connected clients.

### Phase 3: Python Risk Engine & AI Agent
- Create a FastAPI worker that listens to the Redis queue.
- Implement deterministic checks (e.g., rapid velocity limits, high-amount thresholds).
- Connect the free-tier Gemini API using structured JSON output to draft standard compliance summaries when a risk threshold is breached.
- Persist all results back into the PostgreSQL database.

### Phase 4: Angular Real-Time Dashboard
- Scaffold a new Angular application utilizing Tailwind CSS.
- Connect to the ASP.NET Core WebSocket stream.
- Build a dual-view interface:
  1. A live-updating monitoring ledger.
  2. A "Deep Dive" panel to read the AI-generated compliance reports and manually override or approve them.
